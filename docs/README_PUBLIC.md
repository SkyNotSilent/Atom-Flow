# AtomFlow（原子流）- 对外发布说明

> 让每一篇看过的知识，都成为可复用的创作资产。

## 一句话介绍

AtomFlow 是面向内容创作者、产品经理和知识工作者的知识工作台，把「今日推送 → 知识沉淀 → 无限画布创作」连接成一个可追溯的工作流。

## 核心场景

- **高质量输入**：聚合 RSS 和公开内容来源，快速进入全文阅读。
- **知识沉淀**：保存原文，并通过 AI 或规则回退拆解为观点、数据、金句和故事卡片。
- **内容复用**：按类型、来源和引用上下文组织个人知识资产。
- **画布创作**：在魔法写作无限画布中连接文章、原子卡、笔记、文本、文件、图片和写作 Agent。

## 产品能力

### 今日推送

- 多来源内容流和自定义 RSS 订阅
- 全文抓取、原文阅读与来源回溯
- 一键原子化存入知识库

### 我的知识库

- 观点、数据、金句、故事四类原子卡片
- 按类型、来源和标签筛选
- 卡片关联保存的原文和引用上下文
- 支持手动创建、编辑和删除

### 魔法写作无限画布

- 基于 tldraw 的可缩放、可平移无限画布
- 将收藏文章、原子卡、笔记、粘贴文本、文件和图片作为可见节点
- 通过连线明确哪些节点会作为某个写作 Agent 的上下文
- 每个 Agent 可配置模型、系统提示词、`temperature`、`top_p` 和最大输出长度
- 与 Agent 对话后，手动把选中的结果保存回画布，保留创作控制权
- 画布项目、节点、连线、位置、对话和结果持久化到 PostgreSQL

生产环境启用画布需要符合 tldraw 许可证并配置适当的 `VITE_TLDRAW_LICENSE_KEY`。AtomFlow 的 MIT License 不会重新许可 tldraw，详见[第三方声明](../THIRD_PARTY_NOTICES.md)。

## AI 与外部服务

默认 AI 路径使用 MiMo Token Plan 的 OpenAI-compatible API；部署者也可配置官方 OpenAI 或其他兼容提供商。可选能力包括百度翻译、火山引擎实时语音识别、Resend/SMTP 邮件，以及 Railway/PostgreSQL 托管。

这些功能可能把用户选择的正文、提示词、画布上下文、图片、翻译文本、实时音频或邮箱发送给相应供应商。具体实例必须在功能发生前提供就地告知，并公开实际供应商、地区和保留规则。详见[隐私说明](../PRIVACY.md)。

## 适用人群

- 需要持续阅读、沉淀和输出的内容创作者
- 需要保留信息来源与推理上下文的产品经理
- 希望把收藏转成可组合知识资产的知识工作者

## 重要边界

- AI 输出可能不准确，发布前应核验事实、引用和权利。
- RSS 和网页内容仍属于其权利人，保存或引用不等于获得再发布授权。
- 不要上传无权处理的个人信息、商业秘密或第三方私密材料。
- 本项目不声称通过任何安全、隐私或合规认证。
