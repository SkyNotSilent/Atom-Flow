# AtomFlow 隐私说明

版本：1.0｜最后更新：2026-07-12

本说明以中文为主要版本，说明 AtomFlow 开源软件在当前实现下可能处理的数据。它不是任何未指明主体的隐私承诺，也不代表法律合规认证。

## 1. 谁负责你的数据

AtomFlow 是可自行部署的软件。对某个线上实例而言，决定处理目的和方式的是该实例的**部署运营者**，不是仅因提交代码而参与项目的开源贡献者。

每个公开实例必须在用户注册或提交个人信息前填写并展示：

- 运营者名称：`[DEPLOYMENT_OPERATOR_NAME]`
- 联系地址/注册地：`[DEPLOYMENT_OPERATOR_ADDRESS]`
- 隐私联系人：`[PRIVACY_CONTACT_EMAIL]`
- 安全联系人：`[SECURITY_CONTACT_EMAIL]`
- 服务域名：`[SERVICE_URL]`
- 主要存储地区：`[DATA_HOSTING_REGION]`
- 日志保留期：`[LOG_RETENTION_DAYS]`
- 备份保留期：`[BACKUP_RETENTION_DAYS]`
- 数据请求处理时限：`[RIGHTS_REQUEST_RESPONSE_DAYS]`

占位符未填写时，本文件只能作为开源部署模板，不应被表述为某个实际服务已经完成的隐私告知。

## 2. 处理哪些数据及用途

| 类别 | 示例 | 主要用途 |
| --- | --- | --- |
| 账号与验证 | 邮箱、昵称、密码哈希、验证码摘要、头像 | 注册、登录、找回密码、账号展示和安全验证 |
| 会话与安全 | 会话标识、IP 地址、请求时间、请求路径、错误及限流事件 | 保持登录、阻止滥用、排障和安全审计 |
| 订阅与阅读 | RSS 地址、来源名称、文章 URL、标题、正文、图片 URL、音频 URL | 聚合订阅、抓取全文、展示和保存文章 |
| 知识资产 | 收藏原文、原子卡片、标签、引用上下文、笔记、卡片关系 | 建立、检索和管理个人知识库 |
| 写作与 AI | 提示词、对话、选中的上下文、Agent 配置、模型输出、反馈和运行摘要 | 原子化、检索、生成草稿、保存写作历史和排障 |
| 无限画布 | 项目、节点、连线、位置、上传文件/图片、文件名、MIME 类型、提取文本 | 提供可持久化的创作画布和上下文连接 |
| 语音 | 实时音频分片、识别文本、会话技术元数据 | 将语音实时转换为文字输入 |
| 邮件投递 | 收件邮箱、验证码邮件内容、投递和错误元数据 | 发送登录、注册或密码重置验证码 |
| 设备与运行日志 | 浏览器/网络请求元数据、服务端请求 ID、性能和错误信息 | 运行服务、诊断故障和保护安全 |

请勿向服务提交不必要的敏感个人信息、商业秘密、第三方私密内容或你无权处理的材料。上传材料可能包含文档元数据或他人的个人信息，提交前应自行检查并取得必要授权。

## 3. 数据从哪里来

数据可能来自：

- 你直接输入、上传、录音、收藏或配置的内容；
- 你添加的 RSS 地址和 AtomFlow 代你访问的公开网页；
- 浏览器、会话 Cookie 和服务端运行日志；
- 受你指示提供 AI、翻译、语音识别或邮件投递的供应商；
- 为安全、故障排查或权利请求而与你的通信。

AtomFlow 不因本软件本身出售个人信息。具体实例如引入分析、广告、支付或本表未列出的供应商，运营者必须先更新告知并完成适用的选择/同意机制。

## 5. 就地告知要求

完整隐私说明不能替代功能发生时的清晰提示。公开部署必须在相关按钮或输入附近，在数据发出**之前**提供简短、可理解的就地告知：

1. **AI 原子化/写作**：说明实际模型提供商、将发送的文章/提示词/所选节点/图片范围、是否保存输入输出，并提供取消入口。
2. **翻译**：在首次或每次敏感文本翻译前说明文本将发送给百度翻译或实际配置的翻译商。
3. **语音识别**：在开始录音前说明麦克风音频将实时传给火山引擎或实际 ASR 商；停止按钮必须明显，浏览器权限不等同于完整告知。
4. **文件和图片上传**：说明允许的文件类型、存储位置、保留方式，以及文件何时可能作为 AI 上下文发送。
5. **验证码邮件**：在收集邮箱和点击发送前说明邮件服务商及用途，不将验证邮箱默认用于营销。
6. **自定义 RSS/网页抓取**：说明服务器会访问用户填写的 URL，并提醒不要提交内网地址、含密钥 URL 或无权抓取的来源。
7. **供应商或地区变更**：在变更生效前更新本说明；若适用法律要求，应重新取得单独同意。

需要同意时，应让同意与服务条款接受相分离、默认不勾选、可以撤回，并记录版本和时间。拒绝可选处理不应阻止使用与该处理无关的基础功能。

## 6. Cookie 和会话

AtomFlow 使用必要的会话 Cookie 维持登录和防止未授权访问。当前服务端配置的浏览器会话有效期为 7 天，并在生产环境使用 `httpOnly`、`secure` 和 `sameSite=lax` 设置。具体实例若增加分析、广告或非必要 Cookie，运营者必须单独披露并在需要时取得选择。

## 7. 保留与删除

当前代码支持在个人设置中导出账户数据并自助注销账户；注销会删除用户记录，并通过数据库级联删除关联的知识库、写作和画布数据，同时清理验证码和服务端会话。运营者仍需确保备份轮换、第三方副本和日志按本说明处理。

| 数据 | 当前/要求的保留规则 |
| --- | --- |
| 验证码 | 10 分钟后失效；已使用或过期记录由服务端定时清理，最长额外保留 24 小时。 |
| 会话 | 浏览器 Cookie 最长 7 天；服务端会话记录按会话存储清理策略删除。 |
| 账号与用户内容 | 通常保留到用户删除具体内容、账号删除请求完成，或服务终止且不再需要。 |
| 原始语音 | 当前 AtomFlow 代理仅在内存中转实时分片，连接结束后释放，不写入应用数据库；第三方 ASR 的处理以其政策和运营配置为准。 |
| 请求/安全日志 | `[LOG_RETENTION_DAYS]` 天；应以排障和安全所需的最短期限为准，并避免记录正文、密钥和完整验证码。 |
| 备份 | `[BACKUP_RETENTION_DAYS]` 天后轮换或删除；删除请求完成后，数据可能在受限备份中保留到轮换结束，但不得恢复作日常用途。 |
| 第三方副本 | 由实际供应商的合同、数据控制和法律义务决定；运营者应使用可用的最短保留/零保留设置。 |

运营者必须在公开服务前填写日志、备份和权利请求期限，并建立可执行的删除作业。为履行法律义务、解决争议或防止滥用而需要延长保留时，应限制访问并记录理由。

## 8. 你的选择和权利

根据适用法律和具体处理关系，你可以请求：

- 确认是否处理并访问你的数据；
- 更正不准确的账号资料；
- 获取可移植副本；
- 删除账号或指定内容；
- 限制或反对某些处理；
- 撤回基于同意的可选处理；
- 了解第三方接收方、跨境处理和自动化处理情况；
- 向有权机关投诉。

访问、导出、更正和删除可优先通过个人设置自助完成；无法自助完成的请求请联系 `[PRIVACY_CONTACT_EMAIL]`。运营者可以在必要范围内验证请求者身份，并应在 `[RIGHTS_REQUEST_RESPONSE_DAYS]` 天内或适用法律要求的期限内答复。请勿通过公开 Issue 提交身份证件、账号数据或隐私请求。

## 9. 跨境处理

MiMo、OpenAI-compatible 提供商、Railway、Resend、tldraw 或其他供应商可能在用户所在国家/地区之外处理数据。运营者必须在具体实例中披露实际数据路径、地区、法律机制和供应商，并在适用时取得单独同意或提供替代功能。本仓库不保证任何特定数据驻留区域。

## 10. 安全

我们使用合理的技术措施降低风险，但互联网传输和电子存储无法保证绝对安全。部署要求和漏洞报告方式见 [SECURITY.md](SECURITY.md)。用户应使用独立强密码，不在提示词、RSS URL 或上传文件中包含不必要的密钥。

## 11. 未成年人

AtomFlow 不专门面向未满 14 周岁的儿童。若具体部署允许未成年人使用，运营者必须按当地法律设置适龄规则、监护人同意和儿童信息保护措施。发现未经适当授权处理儿童数据时，请联系隐私联系人。

## 12. 变更

重要变更应通过服务内通知、邮件或其他可触达方式提前说明，并标注更新日期。若变更需要新的同意，运营者应在继续相关处理前取得同意。Git 仓库中文件的更新本身不代表所有第三方部署已同步采用。
